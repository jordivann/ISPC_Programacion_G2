<h3>Problemática</h3>
<p>Muchos comercios, depósitos o pequeñas empresas gestionan su stock de forma manual, usando planillas o registros poco organizados. Esto genera errores en el control de productos, falta de trazabilidad sobre entradas y salidas, desconocimiento del stock real, compras innecesarias, faltantes de mercadería y dificultad para tomar decisiones. El proyecto propone diseñar una base de datos relacional para registrar productos, proveedores, categorías, depósitos, movimientos de stock y ventas/compras, permitiendo luego analizar la información con Python para detectar productos críticos, rotación de inventario y necesidades de reposición.</p>

<h5>Qué se puede resolver con el sistema</h5>
<p>Un sistema de stock puede responder preguntas como:</p>
<ul>
    <li>
        ¿Qué productos están por debajo del stock mínimo?
    </li>
    <li>
        ¿Qué productos tienen mayor rotación?
    </li>
    <li>
        ¿Qué productos casi no se mueven?
    </li>
    <li>
        ¿Cuánto stock hay valorizado en dinero?
    </li>
    <li>
        ¿Qué proveedor abastece más productos?
    </li>
    <li>
        ¿Qué categoría representa mayor inversión?
    </li>
    <li>
        ¿En qué fechas hubo más movimientos de entrada o salida?
    </li>
    <li>
        ¿Qué productos deberían reponerse primero?
    </li>
    <li>
        ¿Hay diferencias entre stock registrado y movimientos históricos?
    </li>
    <li>
        ¿Qué depósito o sucursal concentra más mercadería?
    </li>
    
</ul>
